# 認証 概要

## 目的・背景

タスクデータをユーザごとに分離して管理するため、ユーザ識別の仕組みが必要。Supabase Auth を利用してメールアドレス＋パスワードによる認証を実装する。

## スコープ

### 作るもの

- メールアドレス＋パスワードによるユーザ登録
- ログイン・ログアウト
- ログイン状態の保持（セッション管理）
- 未ログイン時のルートガード

### 作らないもの

- ソーシャルログイン（Google・GitHub など）
- パスワードリセット（後回し）
- メール認証（確認メール送信）

## 制約

- Supabase Auth の機能範囲内で実装する
- パスワードポリシーは Supabase のデフォルト設定に従う（8文字以上）

## 完了条件

- 新規ユーザがメールとパスワードで登録できる
- 登録済みユーザがログインしてタスク一覧画面に遷移できる
- ログアウト後にタスク一覧画面へのアクセスがログイン画面にリダイレクトされる
