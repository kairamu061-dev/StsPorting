# 認証 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | ユーザ登録 | メールアドレスとパスワードで新規アカウントを作成する |
| 2 | ログイン | 登録済みアカウントでセッションを開始する |
| 3 | ログアウト | セッションを終了してログイン画面に戻る |
| 4 | ルートガード | 未認証ユーザを保護ルートからログイン画面にリダイレクトする |

## 画面・操作フロー

1. 未ログインユーザが `/tasks` にアクセスする
2. `/login` にリダイレクトされる
3. ログイン画面で「アカウントを作成」を押すと `/register` に遷移する
4. メールアドレス・パスワードを入力して「登録」を押す
5. 登録成功後、`/tasks` にリダイレクトされる
6. 次回以降は `/login` からメールアドレス・パスワードを入力してログインする

## 各画面 / 状態の詳細

### ログイン画面（`/login`）

- **表示内容**: メールアドレス入力欄、パスワード入力欄、ログインボタン、「アカウントを作成」リンク
- **操作**: フォーム送信 → Supabase Auth でセッション発行
- **遷移先**: 成功時 → `/tasks`、失敗時 → 同画面にエラー表示

### 登録画面（`/register`）

- **表示内容**: メールアドレス入力欄、パスワード入力欄、登録ボタン、「ログインに戻る」リンク
- **操作**: フォーム送信 → Supabase Auth でユーザ作成＋セッション発行
- **遷移先**: 成功時 → `/tasks`、失敗時 → 同画面にエラー表示

## エラーケース

| 条件 | メッセージ / 挙動 |
|------|------------------|
| メールアドレスの形式が不正 | 「有効なメールアドレスを入力してください」 |
| パスワードが8文字未満 | 「パスワードは8文字以上で入力してください」 |
| 登録済みメールアドレスで登録 | 「このメールアドレスはすでに使われています」 |
| メールアドレスまたはパスワードが不一致 | 「メールアドレスまたはパスワードが正しくありません」 |
| ネットワークエラー | 「通信エラーが発生しました。再度お試しください」 |

## 未対応ケース

- ソーシャルログインは対象外
- パスワードリセットは将来対応
- メール確認フローは省略（Supabase の confirm email 設定を無効にする）
