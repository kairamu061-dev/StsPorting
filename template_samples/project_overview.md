# プロジェクト概要

## 目的・背景

個人・チーム向けのタスク管理アプリ。既存のツールは機能過多で学習コストが高いため、シンプルな操作でタスクの登録・管理・完了確認ができるWebアプリを作る。

## スコープ

### 作るもの

- ユーザ登録・ログイン（メールアドレス＋パスワード）
- タスクのCRUD（作成・一覧・編集・削除）
- タスクへのラベル付け・フィルタリング
- 完了率のダッシュボード表示

### 作らないもの

- チーム共有・コラボレーション機能
- モバイルアプリ（Webのみ）
- 外部カレンダー連携
- 通知・リマインダー

## 技術スタック概要

| 技術 | 用途 | 選定理由 |
|------|------|----------|
| Next.js (App Router) | フロントエンド | SSRとCSRを柔軟に使い分けられる |
| Supabase | 認証・DB | 認証とPostgreSQLをまとめて扱える |
| Tailwind CSS | スタイリング | 素早くUIを構築できる |
| Vercel | ホスティング | Next.jsとの親和性が高い |

## 全体アーキテクチャ

```
ブラウザ
  └── Next.js (Vercel)
        ├── App Router (Server Components)
        └── Supabase Client
              ├── Auth (JWT)
              └── PostgreSQL (RLS で行レベル制御)
```

## 制約

- Supabase 無料枠の範囲で運用する（DB 500MB、MAU 50,000）
- 初期リリースはソロ開発のため、実装工数を最小化する

## UI デザイン方針

### コンセプト

「余白と整列」を重視したミニマルデザイン。余計な装飾を排除し、タスクの内容そのものに集中できるUIを目指す。操作に迷わない直感的な導線を優先する。

### カラーパレット

| 用途 | カラーコード | 説明 |
|------|-------------|------|
| プライマリ | `#6366F1` | インディゴ。ボタン・リンク・フォーカスリングに使用 |
| セカンダリ | `#E0E7FF` | 薄いインディゴ。バッジ・ハイライトに使用 |
| 背景 | `#F9FAFB` | ほぼ白。ページ全体の背景 |
| テキスト | `#111827` | ほぼ黒。本文テキスト |
| エラー | `#EF4444` | 赤。エラーメッセージ・バリデーション |

### タイポグラフィ

- フォント: `Inter`（システムフォントフォールバック: `sans-serif`）
- 本文: 14px / line-height 1.6
- 見出し: 20px（h1）、16px（h2）、14px bold（h3）

### コンポーネント方針

- **ボタン**: 角丸 `rounded-md`、プライマリは塗り・セカンダリはアウトライン
- **フォーム**: ラベルは入力欄の上、エラーは入力欄の下に赤字で表示
- **カード**: 白背景＋`shadow-sm`＋`rounded-lg`、ホバーで `shadow-md`
- **間隔**: 基本単位 4px（Tailwind の `space-*` に準拠）
