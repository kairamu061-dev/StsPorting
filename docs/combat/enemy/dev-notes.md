# enemy 開発メモ

## 実装上の判断

| 判断内容 | 理由 |
|----------|------|
| 予告ダメージは DamageAction と同一の修正関数で計算 | 予告と実際に食らう値を一致させる原作挙動を再現。脆弱/敵筋力を反映 |
| 行動 AI を「候補＋重み＋連続上限」の共通枠組みに | 原作の確率テーブルと lastMove 連続制限を表現。各敵の具体は content が与える |
| 行動選択は RunRng で決定的 | 同一シードで同一展開。ヘッドレステスト可能 |
| takeTurn の効果は全て action-queue 経由 | 演出/誘発順をプレイヤー側と統一 |
| 次インテントは行動後に再ロール | 原作の「行動後に次を予告」の流れに一致 |

## 今後の課題

- 原作の各敵の確率テーブル・連続制限の正確な数値は content/monsters で再現対象を選定しつつ詰める
- 召喚/分裂など編成変化を伴う行動の扱い（CombatState の敵リスト操作）は段階対応
- ボス固有 AI（フェーズ変化）は content と連携して実装

## ユーザへの要望

- 第1幕で「この敵は入れてほしい」希望があれば優先実装したい（例: ジョー・ラガヴリン、カルト信者、スライム系、エリート、第1幕ボス など）
