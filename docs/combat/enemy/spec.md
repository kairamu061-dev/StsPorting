# enemy 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | Enemy モデル | Creature 継承（HP/ブロック/パワー）＋インテント・履歴 |
| 2 | インテント | 次行動の種別と予告ダメージ（プレイヤーパワー反映） |
| 3 | 行動 AI | 確率テーブル＋連続制限で次行動を決定（RunRng） |
| 4 | takeTurn | 選択行動を GameAction 列としてキューへ |
| 5 | 次インテント決定 | 行動後に次行動を再ロールしインテント更新 |
| 6 | 行動履歴 | 直近行動の保持と連続回数判定 |
| 7 | 初期化 | 配置時に HP ロール・初期インテント決定 |

## インテント種別

| 種別 | 表示 | 意味 |
|------|------|------|
| ATTACK | ⚔＋数値 | 単発攻撃。数値は実食らうダメージ |
| ATTACK_MULTI | ⚔＋数値x回数 | 連続攻撃 |
| BLOCK / DEFEND | 🛡 | 防御（自ブロック獲得） |
| BUFF | ↑ | 自己強化（筋力等） |
| DEBUFF | ↓ | プレイヤーにデバフ |
| ATTACK_DEFEND 等 | 複合 | 攻撃＋防御など |
| UNKNOWN | ？ | 予告不能/特殊 |

予告ダメージは `表示値 = applyPlayerVulnerable(applyEnemyStrength(baseDamage))`（実際に食らう値）。攻撃回数も併記。

## 行動 AI の枠組み

```
getNextMove(rng, history):
  1. 候補リスト（行動ID, 重み, 連続上限）を取得（content が定義）
  2. history を見て「連続上限に達した行動」を候補から除外
  3. 残り候補の重みで rng 抽選
  4. 選んだ行動をインテントに変換し、history を更新
```

- 「連続上限」：同一行動の連続回数制限（例：攻撃は3連続まで、特定スキルは2連続まで）。原作の `lastMove`/`lastMoveBefore` 判定を再現。
- 決定的：同一シード・同一履歴で同一結果。

## takeTurn の流れ

1. turn-flow の敵ターンで各敵の `takeTurn()` が呼ばれる
2. 現在インテントに対応する GameAction を順にキューへ（例：ATTACK→DamageAction、DEFEND→GainBlockAction、BUFF→ApplyPowerAction）
3. 行動完了後 `rollNextMove()` で次インテントを決定・表示更新

## エラーケース

| 条件 | 挙動 |
|------|------|
| 候補が全て連続上限で除外 | フォールバック（上限を一時緩和 or 既定行動）。content で各敵に既定を定義 |
| ターン中に敵が死亡 | 残りの自身の行動を打ち切り |
| 予告ダメージ計算でプレイヤー死亡済み | 表示更新のみ（戦闘終了側で処理） |
| 複数体の行動順 | 配置順（左から/インデックス順）で解決。turn-flow が順に takeTurn |

## 未対応ケース

- 各敵の具体テーブル・特殊ギミック（content/monsters・ボス固有）
- 召喚/分裂など編成変化を伴う行動は段階対応
- スプライト/アニメーション（effects）
