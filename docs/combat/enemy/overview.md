# enemy 概要

## 目的・背景

敵クリーチャーのモデル・インテント（次行動の予告）・行動 AI（行動選択ロジック）を担う。原作の「敵は次に何をするかをアイコンで予告し、確率と連続行動制限に基づいて行動を選ぶ」挙動を再現する。

敵の具体（HP・行動パターン・どのエネミー）は content/monsters が定義するが、本サブ項目は**敵という仕組み**（Creature 派生・インテント表現・行動選択の枠組み・takeTurn の実行）を提供する。

## スコープ

### 作るもの

- **Enemy（AbstractMonster）**：Creature を継承（HP/ブロック/パワー）＋インテント・行動履歴
- **インテント表現**：次行動の種別（攻撃/防御/バフ/デバフ/複合/不明）と、攻撃時の予告ダメージ値（プレイヤーのパワー込みで表示）
- **行動 AI の枠組み**：`getNextMove(rng, history)` による確率・連続制限付きの行動選択
- **takeTurn**：選択済み行動を GameAction としてキューへ（attack/block/applyPower 等の組み合わせ）
- **次インテント決定**：行動後に次ターンの行動を再ロールしインテントを更新
- **行動履歴**：直近の行動を保持し「同じ行動の連続回数制限」を判定

### 作らないもの

- 各敵の具体パターン・数値（content/monsters）
- 敵編成（エンカウンター）の決定（content/run）
- 敵スプライト/アニメーション（effects）

## スコープ補足（再現方針）

原作の敵 AI は敵ごとに固有の確率テーブルと「同一行動の連続回数制限」を持つ。本サブ項目は AI を表現する共通枠組み（行動候補・重み・制限）を提供し、content/monsters が各敵の具体テーブルを与える。

## 制約

- 行動選択は RunRng（combat 系統）で決定的に行い、ヘッドレスでテスト可能にすること。
- インテントの予告ダメージは、プレイヤー側のパワー（脆弱等）と敵側の筋力を反映した「実際に食らう値」を表示する（原作準拠）。
- takeTurn の効果は全て action-queue 経由（即時適用しない）。

## 完了条件

- 敵が HP/インテントを持って配置され、次行動を予告する
- takeTurn で予告どおりの行動がキューに積まれ解決する
- 確率・連続制限に従って次行動が決定的に選ばれる（同一シードで同一）
- インテントの予告ダメージがプレイヤーのパワーを反映する
- 行動選択・予告計算がヘッドレステストで通る

## 画面イメージ

敵の頭上にインテントアイコン（描画は effects、論理はここ）：

```
   [⚔ 12]        [🛡]         [⚔⚔ 6x2]      [↑]
   ジョー        防御態勢      連撃           バフ
   HP 40/40     HP 28/28     HP 45/45      HP 30/30
```

行動選択の概念：

```
getNextMove(rng, history):
  候補=[攻撃A(50%), 防御(30%), デバフ(20%)]
  連続制限: 攻撃Aは3連続まで, デバフは2連続まで
  → 制限に触れる候補を除外し、残り重みで rng 抽選
```
