# powers 開発メモ

## 実装上の判断

| 判断内容 | 理由 |
|----------|------|
| フックは AbstractPower の既定 no-op＋必要分のみ override | 原作 AbstractPower のモデル。各パワーが関係するフックだけ実装でき拡張容易 |
| ダメージ修正順を DamageAction 側で固定（give→receive→block→hp→onAttacked） | 筋力/弱体/脆弱/ブロック/とげの相互作用結果を原作と一致させる根幹 |
| 乗算後に切り捨て（floor） | 弱体0.75・脆弱1.5 の端数処理を原作に合わせる。順序で結果が変わるため明文化 |
| 同種パワーは amount 加算で 1 インスタンス | 重複インスタンスを作らず、原作のスタック表現に一致 |
| デバフは 0 で消滅、筋力は負値許容 | 弱体/脆弱の自然減衰と、筋力低下（負の筋力）を両立 |
