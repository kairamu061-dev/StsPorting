# powers 概要

## 目的・背景

クリーチャーに付与されるパワー（バフ/デバフ・状態異常）の仕組みを担う。筋力・弱体・脆弱・中毒・金属化・とげ等、ダメージ計算やターン境界に干渉する効果の付与・スタック・減衰・発動を、原作のフックモデルで再現する。

パワーは action-queue・turn-flow・cards・enemy のあらゆる箇所から参照され、ダメージ修正や被弾誘発の中心。本サブ項目は**パワーという仕組み**（フック点とスタック規則）を提供し、個々のパワーの一部はここで、コンテンツ寄りのものは content と連携して定義する。

## スコープ

### 作るもの

- **AbstractPower**：パワーの抽象基底（amount＝スタック、種別、各種フック）
- **フック群**：ダメージ修正・被ダメ修正・被弾時・与ダメ時・ターン頭/終わり・カードプレイ時・ブロック時など、原作準拠の発火点
- **スタック規則**：同種パワーの加算、上限、0 で消滅、ターン経過での減衰（弱体/脆弱は毎ターン-1 等）
- **コアパワーの実装**：筋力(Strength)/弱体(Weak)/脆弱(Vulnerable)/中毒(Poison)/金属化(Metallicize)/とげ(Thorns)/再生(Regen)/筋力低下(一時) など、アイアンクラッド戦で頻出する基本パワー
- **付与/減少アクション連携**：ApplyPowerAction / ReducePowerAction（action-queue）からの操作

### 作らないもの

- ダメージ計算ループ本体（action-queue の DamageAction が、パワーの修正フックを呼ぶ）
- パワーアイコンの描画/数値演出（effects。論理値と更新通知のみ）
- カード/レリック固有の特殊パワー全種（content と連携し代表を実装）

## 制約

- フックの呼び出し順を明確に定義し、ヘッドレスで決定的にテストできること。
- パワーの減衰・消滅のタイミングを turn-flow のフック発火順と一致させる。
- DamageAction の計算順（攻撃修正→被ダメ修正→ブロック→HP→被弾フック）に正しく組み込まれること。

## 完了条件

- 筋力で攻撃力が加算、弱体で攻撃 0.75 倍、脆弱で被ダメ 1.5 倍が正しく適用される
- 中毒がターン頭に HP を削りスタックが 1 減る、とげが被弾時に反撃する
- 同種パワー付与でスタックが加算され、0 で消滅する
- 弱体/脆弱がターン終わり（または規定タイミング）に -1 される
- 上記がヘッドレステストで決定的に通る

## 画面イメージ

クリーチャー下のパワーアイコン列（描画は effects、論理はここ）：

```
[敵] HP 40/40  ブロック8
  🟥筋力3  🟪脆弱2  🟢中毒5     ← amount を併記したアイコン列
```

フック発火の概念：

```
DamageAction:
  base=6
  → atDamageGive(筋力) → 6+3=9
  → atDamageGive(弱体)  → 9*0.75=6
  → atDamageReceive(脆弱) → 6*1.5=9
  → ブロック軽減 → HP 減算
  → onAttacked(とげ) → addToTop(反撃ダメージ)
```
