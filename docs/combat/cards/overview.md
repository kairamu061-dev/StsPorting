# cards 概要

## 目的・背景

戦闘中のカードの実体・所在（山札/手札/捨て札/消滅札）・プレイ処理を担う。カードは戦闘の主たる操作対象であり、その挙動（ドロー、コスト、対象、効果の発火、使用後の移動先）を原作どおりに再現する。

カードの「効果そのもの」（攻撃6/ブロック5 等）は content/cards が定義するが、本サブ項目は**カードという仕組み**（種別・コスト・対象・piles・プレイのライフサイクル）を提供する。content はこの上で各カードを表現する。

## スコープ

### 作るもの

- **CardInstance**：戦闘中の 1 枚（定義への参照＋戦闘内可変状態：コスト一時変更・アップグレード・一時効果）
- **カード種別**：ATTACK / SKILL / POWER と、それぞれの基本挙動の差
- **対象種別**：単体敵 / 全体 / 自分 / 対象なし（ターゲティングと連携）
- **piles**：drawPile / hand / discardPile / exhaustPile の保持と移動 API
- **ドロー処理**：山札からの引き、山札切れ時の捨て札シャッフル、手札上限（10）
- **プレイのライフサイクル**：プレイ可否判定（コスト/プレイ可能条件）→ コスト消費 → `use()` で効果アクション投入 → 使用後の移動（捨て札/消滅/POWER は消費）
- **アップグレード**：戦闘内アップグレード（一部効果）と表示差分の反映
- **コスト変動**：一時的コスト変更（0 コスト化等）、X コスト、プレイ不可カード（呪い/状態異常カード）

### 作らないもの

- 各カードの具体効果・数値（content/cards）
- ドラッグ/対象選択の入力 UI（input）
- カード移動の演出（effects。pile 移動の論理のみ）

## 制約

- pile 操作・ドロー・プレイ判定はヘッドレスで決定的にテスト可能にする（シャッフルは RunRng）。
- プレイは action-queue 経由（CardQueueItem）。即時に効果を適用せず、コスト消費後にカードの `use()` がアクションを積む。
- 手札上限・山札切れシャッフル等の境界を原作準拠で再現。

## 完了条件

- ドロー/捨て/消滅/シャッフルの pile 移動が正しく動作する
- カードのプレイ可否（コスト・条件）が判定され、プレイで効果アクションが積まれる
- POWER カードはプレイ後に消費され、ATTACK/SKILL は捨て札（または消滅）へ移る
- 山札切れ時に捨て札がシャッフルされ山札へ戻る
- 手札上限を超えるドローが上限で止まる
- 上記がヘッドレステストで通る

## 画面イメージ

手札と pile カウンタ（描画は input/effects と連携、ここでは論理）：

```
山札:7                                          捨て札:3   消滅:1
        \   [Strike][Defend][Bash][Strike+]   /     ← 手札（最大10）
カードのライフサイクル:
  drawPile → hand → (プレイ) → discardPile / exhaustPile
                              POWER は盤面適用後に消費(どこにも残らない)
```
