# input 概要

## 目的・背景

戦闘中のプレイヤー操作（カードのドラッグ＆ドロップ、対象選択、ホバー、ポーション使用、ターン終了）を担う。本プロジェクトの最優先要件「**操作性の完全再現**」が直接現れる領域で、原作のカード操作の手触り（弧状手札、ホバー拡大、ドラッグの追従、対象指定の矢印、ドロップ判定）を忠実に再現する。

ロジック（cards のプレイ判定や action-queue）には触れず、「ユーザ入力を戦闘ロジックへの要求に変換する」境界を担う。演出（拡大・矢印描画）は effects と連携するが、入力状態機械はここが持つ。

## スコープ

### 作るもの

- **手札レイアウト**：弧状（扇形）に並ぶカードの座標計算、枚数に応じた間隔・角度・重なり
- **ホバー**：カーソル下のカードを判定し拡大・前面化（原作の持ち上がり挙動）
- **ドラッグ＆ドロップ**：カードを掴んで持ち上げ、対象指定カードは敵へドラッグで対象矢印、ドロップ位置で確定/キャンセル
- **対象選択**：単体対象（敵へドロップ）、対象なし（プレイ閾値より上へドラッグで発動＝原作の「上方向に離す」）、全体/自分の扱い
- **ポーション使用**：ポーションスロットのクリック→（対象要否に応じ）対象選択→使用要求
- **ターン終了**：ボタン/キーでの終了要求（turn-flow の requestEndTurn）
- **入力可否制御**：action-queue がアイドルかつプレイヤーターンのときのみ操作受付
- **キーボード補助**：数字キーでカード選択、E でターン終了 等（任意・原作補助操作）

### 作らないもの

- カードのプレイ可否判定・効果（cards/action-queue）
- 矢印やカード拡大の描画実体（effects。座標と状態はここ）
- 戦闘外の入力（foundation の InputRouter / 各画面）

## 制約

- foundation の `InputConsumer`（仮想座標）を用い、解像度非依存にヒットテストする。
- 入力状態（IDLE/HOVER/DRAGGING/TARGETING）を明確な状態機械で管理し、原作の操作感（誤爆しにくいドロップ閾値、対象なしカードの発動方法）を再現。
- 操作の最終的な手触りは Windows 実機で反復確認する。

## 完了条件

- 手札が弧状に並び、ホバーで対象カードが拡大・前面化する
- カードをドラッグして持ち上げ、敵へドロップで対象指定カードがプレイ要求される
- 対象なしカードは上方向ドラッグ（閾値超え）で発動、閾値未満リリースでキャンセル
- ポーション使用とターン終了が要求として発火する
- action-queue 解決中は操作を受け付けない
- 入力状態機械のロジックがテストできる（ヒットテスト/状態遷移）

## 画面イメージ

```
状態機械:
  IDLE --hover--> HOVER --press--> DRAGGING
   ^                                  |
   |              対象必要?           |
   |        no: 上方向閾値超でplay     | yes: 敵上でドロップ
   |        no: 閾値未満でcancel       |        -> play(target)
   +----------- release --------------+

手札（弧状・ホバーで持ち上がり）:
        [ Strike ]   ← ホバー中は拡大・前面
   [Defend]      [Bash]      [Strike]
```
