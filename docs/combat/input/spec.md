# input 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | 手札レイアウト | 弧状配置の座標/角度/間隔を枚数から算出 |
| 2 | ホバー判定 | カーソル下カードを判定し拡大・前面化 |
| 3 | ドラッグ | カードを掴んで追従、状態 DRAGGING |
| 4 | 対象選択 | 単体は敵へドロップ、対象なしは上方向閾値、全体/自分の扱い |
| 5 | プレイ要求 | 確定時に cards/action-queue へ CardQueueItem を渡す |
| 6 | ポーション使用 | スロットクリック→対象要否→使用要求 |
| 7 | ターン終了 | ボタン/キーで requestEndTurn |
| 8 | 入力可否 | isPlayerInputAllowed のときのみ受付 |

## 入力状態機械

| 状態 | 説明 | 主な遷移 |
|------|------|----------|
| IDLE | 何も掴んでいない | カード上ホバー→HOVER |
| HOVER | カードにカーソル（拡大表示） | 押下→DRAGGING / 離脱→IDLE |
| DRAGGING | カードを掴んで追従 | リリースで判定（下記） |
| TARGETING | 対象指定カードのドラッグ中（矢印表示） | 敵上リリース→play / それ以外→cancel |

### リリース時の確定ルール（原作準拠の手触り）

- **対象なしカード（自分/全体/NONE）**: カードを「プレイ閾値ライン」より上へドラッグしてリリース → プレイ。閾値未満でリリース → 手札へ戻す（キャンセル）
- **単体対象カード（ENEMY）**: ドラッグ開始で TARGETING（始点カード→カーソルへ矢印）。敵スプライト上でリリース → その敵を対象に play。敵以外でリリース → キャンセル
- プレイ不可（エネルギー不足/プレイ不可カード）: 掴めるがリリースでキャンセル＋赤フィードバック

## 各操作の詳細

### カードのホバー/ドラッグ

- ホバー：手札カードの当たり判定（弧状配置の各カード矩形を仮想座標で判定）。最前面の 1 枚を選ぶ
- 拡大：ホバーカードを拡大し y を持ち上げ（effects が描画、座標と倍率はここ）
- ドラッグ：押下したカードを掴み、カーソルに追従。対象種別で TARGETING か通常ドラッグかを分岐

### ポーション

- ポーションスロットアイコンをクリック→メニュー（使用/捨てる）。使用で対象要否を判定し、要対象なら敵選択、不要なら即使用要求

### ターン終了

- 「ターン終了」ボタンクリック、または E キー（任意）。requestEndTurn を呼ぶ
- 解決中（非アイドル）は無効

## エラーケース

| 条件 | 挙動 |
|------|------|
| 解決中の入力 | 無視（isPlayerInputAllowed=false）。ボタンも非活性表示 |
| プレイ不可カードのリリース | キャンセル＋フィードバック |
| 対象指定で敵以外にドロップ | キャンセル |
| 複数カード重なりのホバー | 最前面（弧の中で最後に描画される手前）の 1 枚を対象 |
| ドラッグ中にウィンドウ外へ | 仮想座標でクランプ。リリースでキャンセル扱い |

## 未対応ケース

- ゲームパッド操作
- カード以外の高度なドラッグ操作（並び替え等、原作にない要素）
- 矢印/拡大の描画実体（effects）
