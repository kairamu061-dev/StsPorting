# input 開発メモ

## 実装上の判断

| 判断内容 | 理由 |
|----------|------|
| 入力を IDLE/HOVER/DRAGGING/TARGETING の状態機械に | 操作の誤爆を防ぎ、原作の手触り（持ち上げ→対象→確定）を明確に再現 |
| 対象なしカードは上方向の閾値リリースで発動 | 原作の「上に放ると発動・下で戻る」操作を再現し、誤プレイを防ぐ |
| 単体対象は TARGETING＋矢印、敵ドロップで確定 | 原作の対象指定操作の手触りを再現 |
| 入力可否を turn-flow.isPlayerInputAllowed に従わせる | 演出/解決中の操作を遮断し、原作の「演出中は操作不可」を再現 |
| ロジック（座標/状態）と描画（拡大/矢印）を分離 | 入力状態をテスト可能にし、描画は effects に委譲 |

## 今後の課題

- PLAY_THRESHOLD_Y や弧の角度・拡大倍率など「手触りの定数」は実機で原作と見比べて調整
- カード整列/追従の補間速度の調整（なめらかさ）
- ポーション使用 UI（使用/捨てる）の細部は run のポーション管理と擦り合わせ

## ユーザへの要望

- 操作感の最終調整には Windows 実機での反復確認が必要。基準にしたい原作の挙動（録画やプレイ）があれば共有してほしい
