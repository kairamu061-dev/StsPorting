# action-queue 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | キュー解決ループ | 現在アクションを毎フレーム update し、完了で次へ。空になるまで継続 |
| 2 | 末尾追加 addToBottom | 通常の効果追加。既存キューの後ろに積む |
| 3 | 割り込み addToTop | 実行中の効果が誘発した効果を先頭に積み、先に解決させる |
| 4 | 演出待ち | duration を持つアクションは経過時間が満ちるまで完了しない |
| 5 | カードプレイ投入 | CardQueueItem を受け、コスト消費＋カード効果アクションをキューへ |
| 6 | 基本アクション群 | Damage/GainBlock/Draw/Discard/ApplyPower/Exhaust 等の原始処理 |
| 7 | フック発火点 | ターン頭/終わり・カードプレイ後など定常アクションの差し込み |

## 解決ループの詳細

毎フレーム `ActionManager.update(delta)` は以下を行う:

1. `currentAction` が null なら、キュー先頭から 1 つ取り出して `currentAction` にする（空なら何もしない＝アイドル）
2. `currentAction.update(delta)` を呼ぶ
3. アクション内部で `addToTop` された場合、それらは次の取り出しで先に処理される（＝割り込み）
4. `currentAction.isDone == true` になったら `currentAction = null`（次フレームで次を取り出す）
5. キューも currentAction も空になったらアイドル（プレイヤー入力待ち or ターン遷移可能）

### addToTop / addToBottom の順序規約

- `addToBottom(a)`：キュー末尾へ。通常の逐次効果。
- `addToTop(a)`：キュー先頭へ。複数を連続で addToTop した場合、**最後に積んだものが最初に解決**される（先頭に押し込むため）。原作の誘発順に合わせる。
- 1 つのアクションが複数の割り込みを「列として正順で」入れたい場合は、専用のまとめアクション（順序を内部保証）を用いるか、逆順に addToTop する。実装規約として明文化する。

## 基本アクション仕様（抜粋）

| アクション | 効果 | 主な完了条件 |
|-----------|------|------------|
| DamageAction(target, amount, source) | ダメージ計算（攻撃修正→被ダメ修正→ブロック→HP）→被弾フック割り込み | 計算適用＋演出後 |
| GainBlockAction(target, amount) | ブロック付与（筋力等の修正適用後） | 即時/短演出 |
| DrawAction(n) | 山札から n 枚ドロー（山札切れは捨て札シャッフル割り込み） | n 枚処理後 |
| DiscardAction / ExhaustAction | 手札→捨て札/消滅へ移動 | 移動後 |
| ApplyPowerAction(target, power, amount) | パワー付与/スタック（既存なら加算、減衰規則に従う） | 付与後 |
| MakeTempCardAction 等 | 一時カード生成（必要に応じ） | 生成後 |

> 各アクションの厳密な数値・修正順は combat 親 design.md の「解決順序の規約」に従う。

## エラーケース

| 条件 | 挙動 |
|------|------|
| 対象が既に死亡 | 死亡フラグを確認し、無効な後続効果はスキップ（原作準拠で「死亡時に解決中の効果」は規約化） |
| 山札・捨て札とも空でドロー | 何もせず完了（ドロー不能） |
| 無限ループの懸念（誘発の連鎖） | 各誘発は有限回（スタック減算等）で収束する設計。テストで連鎖が停止することを保証 |
| 演出待ち中にゲーム速度変更 | duration を delta スケールで進め、速度設定に追従（任意機能） |

## 未対応ケース

- 演出の中身（パーティクル等は effects）。ここでは「待ち時間」と「演出開始の合図」のみ
- 個別カード/パワー固有の特殊割り込み（各サブ項目が本機構を使って実装）
