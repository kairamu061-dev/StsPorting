# effects 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | エフェクト管理 | 寿命付きエフェクトの生成/更新/描画/破棄 |
| 2 | 数値ポップ | ダメージ/ブロック/回復の数値を浮かせフェード |
| 3 | 被弾演出 | ヒットフラッシュ・キャラ揺れ・斬撃/パーティクル |
| 4 | ブロック演出 | ブロック増加のきらめき・アイコン更新 |
| 5 | 画面シェイク | 攻撃強度に応じたカメラ揺れ |
| 6 | カード移動 | プレイ/ドロー/捨て/消滅の飛翔アニメ |
| 7 | パワーアイコン | バフ/デバフアイコン＋スタック数の描画 |
| 8 | インテント描画 | 敵頭上のインテントアイコン/数値 |
| 9 | 入力連動描画 | ホバー拡大・ドラッグ追従・対象矢印 |

## アクションとの連動

action-queue の各アクションが解決時に「演出要求」を出し、effects が再生する。アクションの `duration` は演出時間と一致させる。

| アクション | 発火する演出 | 目安 duration |
|-----------|-------------|--------------|
| DamageAction | ヒットフラッシュ＋揺れ＋数値ポップ＋（強なら）シェイク | 0.1–0.3s |
| GainBlockAction | ブロックきらめき＋数値ポップ | 0.1s |
| DrawAction | 山札→手札の飛翔（枚数分） | 0.1s/枚 |
| Discard/Exhaust | 手札→捨て札/消滅の飛翔（消滅は専用色） | 0.15s |
| ApplyPowerAction | アイコン出現/スタック増の小演出 | 0.1s |
| LoseHpAction（中毒等） | 数値ポップ（ブロック無視色） | 0.1s |

> 演出は非ブロッキングにもできるが、打撃感に関わる主要演出はアクション duration で待たせ、原作テンポを再現する。

## 数値ポップ仕様

- ダメージ：赤系、被弾キャラ上に出現、上方向へ移動しながらフェード。大ダメージは大きく
- ブロック貫通せず軽減のみ：軽減を示す表現（任意）
- ブロック獲得：青系、自キャラのブロックアイコン付近
- 回復：緑系
- 中毒/直接 HP 減：通常ダメージと区別（紫系など）

## 画面シェイク仕様

- 攻撃ダメージ量に応じた強度（小/中/大）。time 経過で減衰
- 連続ヒットは累積しすぎないようクランプ

## エラーケース

| 条件 | 挙動 |
|------|------|
| アセット欠落 | プレースホルダ（図形/既定パーティクル）で代替し継続 |
| 大量エフェクト同時発生 | 上限を設けて古いものから破棄（パフォーマンス保護） |
| 演出対象が既に消滅（敵死亡） | 最後の位置で再生 or 即時終了（破綻させない） |
| 低スペックでのフレーム落ち | delta 連動で時間ベース進行（コマ落ちしても所要時間は一定） |

## 未対応ケース

- サウンドエフェクト（任意・別途）
- 原作完全一致のパーティクル形状（アセット差し替えで段階的に近づける）
- 過剰な装飾演出（原作にない演出は足さない）
