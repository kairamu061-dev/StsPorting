# effects 開発メモ

## 実装上の判断

| 判断内容 | 理由 |
|----------|------|
| 演出は CombatState を読み取り専用で参照（状態変更しない） | 描画の単方向依存にし、ロジックと演出を疎結合に。テスト容易性も担保 |
| アクションの duration を演出時間と一致させる | 「演出が終わるまで次に進まない」原作テンポ・打撃感を再現する根幹 |
| 主要演出（被弾/シェイク）はブロッキング、軽微なものは非ブロッキング | テンポを保ちつつ過剰な待ちを避ける |
| 初期はプレースホルダ（図形/簡易パーティクル）で形・タイミング再現 | アセット非同梱のため。後で差し替えても構造は不変 |
| エフェクトはプールで管理 | 大量生成時の GC 負荷を避ける |

## 今後の課題

- 各演出の duration・シェイク強度・数値の動きなど「テンポの定数」は実機で原作と見比べて調整
- パーティクル形状の原作一致はアセット差し替えで段階的に
- サウンド連動は任意・後続（視覚と同じ発火点に乗せられる設計にしておく）

## ユーザへの要望

- 打撃感/テンポの基準にしたい原作の挙動（録画等）があれば共有してほしい。実機での反復確認が必要
