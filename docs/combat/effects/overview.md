# effects 概要

## 目的・背景

戦闘の視覚演出（アニメーション・パーティクル・ダメージ数値・画面シェイク・カード移動・パワーアイコン）を担う。本プロジェクトの最優先要件「**エフェクトの完全再現**」が直接現れる領域で、原作の打撃感・テンポ・視覚的フィードバックを忠実に再現する。

action-queue が各アクション解決時に「演出開始の合図」を出し、本サブ項目がその演出を再生する。演出の再生時間（duration）はアクションの待ち時間と連動し、原作の「演出が終わるまで次に進まない」テンポを作る。

## スコープ

### 作るもの

- **エフェクトマネージャ**：寿命付きエフェクトの生成・更新・描画・破棄を一括管理
- **ダメージ/ブロック数値ポップ**：被弾位置に数値を浮かせる（色/サイズ/フェード/上昇）
- **被弾演出**：ヒットフラッシュ、被弾時のキャラ揺れ・点滅、斬撃/パーティクル
- **ブロック獲得演出**：ブロックアイコンの増加・きらめき
- **画面シェイク**：強い攻撃時のカメラ揺れ（強度/時間）
- **カード移動アニメ**：プレイ→捨て札/消滅、ドロー→手札のカード飛翔
- **パワーアイコン表示**：クリーチャー下のバフ/デバフアイコンとスタック数（powers の論理値を可視化）
- **インテントアイコン描画**：敵頭上のインテント（enemy の論理値を可視化）
- **手札の拡大/矢印描画**：input の状態（hovered/dragging/targeting）に応じた描画

### 作らないもの

- 状態変化のロジック（action-queue/powers/cards が担う）
- 入力状態の管理（input。effects は描画のみ）
- サウンド（任意・別途。ここでは視覚演出を主とする）

## 制約

- 演出は CombatState/各サブ項目の論理値を「読むだけ」で、状態を変更しない（描画の単方向依存）。
- 演出時間はアクションの duration と連動し、テンポを原作に合わせる。
- 原作アセット非同梱のため、初期はプレースホルダ（図形/簡易パーティクル）で形・タイミングを再現し、後でアセット差し替え。
- 体感（打撃感/テンポ）は Windows 実機で反復確認する。

## 完了条件

- ダメージ/ブロックの数値ポップが被弾位置に出てフェードする
- 被弾でヒットフラッシュ・揺れ、強攻撃で画面シェイクが起きる
- カードのプレイ/ドロー/捨て札移動がアニメーションする
- パワー/インテントのアイコンが論理値どおりに表示・更新される
- 演出時間がアクションの待ちと連動し、テンポが破綻しない

## 画面イメージ

```
被弾の瞬間:
   [敵] ← 白フラッシュ＋揺れ      "12"（赤数値が上へフェード）
                                  画面が軽くシェイク

ブロック獲得:
   [自] 🛡5 → 🛡10  （+5 のきらめき）

カード移動:
   手札[Strike] --飛翔--> 捨て札パイル
```
