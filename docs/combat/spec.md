# combat 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | 戦闘初期化 | RunState を受け取りデッキを山札へ、敵を配置、開始時パワー/レリックを適用 |
| 2 | アクションキュー | 全効果を逐次実行。割り込みアクションの先頭挿入、解決順を原作準拠で再現 |
| 3 | ターン進行 | プレイヤー → 敵 → の循環。各フェーズの開始/終了フックを発火 |
| 4 | カードプレイ | コスト支払い・対象決定・効果解決・移動先（捨て札/消滅）への遷移 |
| 5 | 山札/手札管理 | ドロー、手札上限、山札切れ時の捨て札シャッフル、消滅 |
| 6 | エネルギー | 毎ターン回復、消費、X コスト、未使用分の繰越なし |
| 7 | パワー | バフ/デバフの付与・スタック・ターン頭/終わりの増減・効果適用 |
| 8 | ダメージ/ブロック | 攻撃修正（筋力±/弱体）→被ダメ修正（脆弱）→ブロック軽減→HP 減算の順 |
| 9 | 敵 AI/インテント | 行動選択（確率・連続制限）と次ターン意図の予告表示 |
| 10 | ターゲティング/入力 | ドラッグでプレイ、対象選択、ホバー拡大、終了ターン、ポーション使用 |
| 11 | エフェクト | 被弾/ブロック/カード移動/数値ポップ/パーティクル/画面シェイク |
| 12 | 勝敗判定 | 敵全滅で勝利、プレイヤー HP0 で敗北。結果を run へ返す |

## 画面・操作フロー

1. `run` が戦闘を開始（`CombatRequest`：敵編成・プレイヤー状態を渡す）
2. 戦闘初期化：デッキをシャッフルして山札化、敵配置、戦闘開始時効果（レリック/パワー）を解決
3. プレイヤーターン開始：エネルギー回復、規定枚数ドロー、ターン頭パワー処理
4. プレイヤーはカードをドラッグしてプレイ（対象が必要なら敵へドロップ）、ポーション使用可
5. 「ターン終了」でプレイヤーターン終了処理（手札保持/破棄、ターン終わりパワー）
6. 敵ターン：各敵が予告済みインテントを実行（アクションキュー経由）
7. 勝敗判定 → 継続なら 3 へ。決着なら `CombatResult` を返して戦闘終了

## 各画面 / 状態の詳細

### プレイヤーターン

- **表示内容**: 手札（弧状）、エネルギー球、山札/捨て札カウンタ、敵インテント、各キャラの HP/ブロック/パワーアイコン
- **操作**:
  - カードをホバー → 拡大表示
  - カードをドラッグ → プレイ。対象指定カードは敵へドロップで対象確定
  - エネルギー不足/プレイ不可カードはドロップで手札へ戻る
  - ポーションをクリック → 使用（対象が要るものは対象選択）
  - 「ターン終了」ボタン or キー → 敵ターンへ
- **遷移先**: ターン終了 → 敵ターン、勝利/敗北 → 戦闘終了

### 敵ターン

- **表示内容**: 敵がインテントに沿って行動するアニメーションと数値
- **操作**: なし（自動進行。アクションキューが解決）
- **遷移先**: 解決後プレイヤーターン or 勝敗判定

### 戦闘終了

- **表示内容**: 勝利時は短い演出 → `run` の報酬へ。敗北時はゲームオーバーへ
- **遷移先**: `run`（CombatResult を返す）

## エラーケース

| 条件 | メッセージ / 挙動 |
|------|------------------|
| エネルギー不足でカードをプレイ | プレイ不可。カードは手札に戻る（赤表示等のフィードバック） |
| 対象指定カードを対象なしでドロップ | キャンセル。手札へ戻る |
| 山札が空でドロー要求 | 捨て札をシャッフルして山札に補充してからドロー。両方空なら何もしない |
| 手札上限（10）超過のドロー | 上限まででドロー停止（原作準拠） |
| ブロック超過ダメージ | ブロックを 0 にし、残りを HP から減算 |
| 同時多数の割り込みアクション | キュー先頭への挿入順を原作準拠で固定し決定的に解決 |

## 未対応ケース

- 全カード/全敵/全パワーの網羅（`content` が再現対象を選定。combat は仕組みを完全再現）
- 特殊ギミック（時間制限・特殊勝利条件など一部ボス固有）は段階対応
- スマート/オートプレイ等の補助機能は作らない

## サブ項目への分割

戦闘は複数の独立システムから成るため、以下に分割する（`/add-feature combat/<name>`）。
親（本ドキュメント）はシステム全体像・戦闘境界（CombatRequest/CombatResult）・解決順序の規約を担う。

- `combat/action-queue` … アクションキュー（解決順序の根幹）
- `combat/turn-flow` … ターン/フェーズ進行・エネルギー・開始/終了フック
- `combat/cards` … カードモデルと山札/手札/捨て札/消滅、プレイ処理
- `combat/powers` … パワー（バフ/デバフ）の付与・スタック・発動
- `combat/enemy` … 敵モデル・インテント・行動 AI
- `combat/input` … ターゲティング・ドラッグ・操作性
- `combat/effects` … アニメーション・パーティクル・数値・画面演出
