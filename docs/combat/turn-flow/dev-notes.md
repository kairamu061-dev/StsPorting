# turn-flow 開発メモ

## 実装上の判断

| 判断内容 | 理由 |
|----------|------|
| フェーズ遷移は actionManager.isIdle 時のみ | 演出/解決の途中で次フェーズに進まない原作挙動を再現。順序の一貫性も担保 |
| 定常処理（シャッフル/ドロー/フック）も全てアクション化 | 即時計算せずキューに積むことで、演出と状態変化の順序を他効果と一致させる |
| ターン終了は requestEndTurn フラグ → 次 isIdle で処理 | 入力と解決を分離し、演出中の終了要求を安全に扱う |
| エネルギーは毎ターン全回復・繰越なし | 原作仕様。X コストは spendAll で現在量を消費 |
| 各遷移点で勝敗チェック | ターン頭自傷死・敵全滅直後など、原作の打ち切りタイミングを再現 |
