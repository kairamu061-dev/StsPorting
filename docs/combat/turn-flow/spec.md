# turn-flow 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | フェーズ状態機械 | BEGIN/PLAYER/END/ENEMY を管理し、キューアイドル時に遷移 |
| 2 | 戦闘開始処理 | 山札シャッフル・敵配置・開始フック・初期ドロー |
| 3 | プレイヤーターン開始 | エネルギー全回復・規定ドロー・ターン頭フック |
| 4 | ターン終了処理 | 手札処理（破棄/保持）・ターン終わりフック・ブロック規則 |
| 5 | 敵ターン | 各敵のインテント実行を順にキュー・敵パワー処理 |
| 6 | エネルギー管理 | 最大値・全回復・消費・X コスト |
| 7 | 勝敗チェック | 各遷移点で敵全滅/プレイヤー死亡を判定 |

## ターン進行の詳細手順

### 戦闘開始（BEGIN）

1. マスターデッキのコピーを山札へ入れ、RunRng でシャッフル
2. 敵を配置（HP ロール・初期インテント決定）
3. `onBattleStart` フック（レリック/開始時パワー）をキューへ
4. 初期ドロー（既定 5 枚）をキューへ
5. キューがアイドルになったら PLAYER へ

### プレイヤーターン開始（PLAYER 突入時）

1. ブロックの持ち越し処理（原則リセット。保持系レリック/パワーは例外）
2. `atTurnStart`（プレイヤー側パワー：再生/筋力付与/燃焼ダメージ等）をキューへ
3. エネルギーを最大まで回復
4. 規定枚数ドロー
5. 入力受付可能（input サブ項目）

### ターン終了（PLAYER → END → ENEMY）

1. プレイヤー操作で「ターン終了」
2. `atTurnEnd`（プレイヤー側パワー：中毒/燃焼の自傷、メタリサイズ等）をキューへ
3. 手札処理：基本は捨て札へ。保持（Retain）系は残す。ターン終わりに作用するカード効果を解決
4. ENEMY へ

### 敵ターン（ENEMY）

1. 各敵について `atTurnStart`（敵パワー）→ `takeTurn`（インテント実行）→ `atTurnEnd` をキューへ（敵の並び順）
2. 行動後、各敵は次インテントを再ロール（enemy サブ項目）
3. キューがアイドルになったら勝敗チェック → PLAYER へ戻る or 終了

## エネルギー仕様

- 最大エネルギー：既定 3（レリック等で変動し得る）
- 毎プレイヤーターン開始で最大まで回復（未使用分は繰り越さない）
- カードプレイ時に `spendEnergy(cost)`。不足ならプレイ不可
- X コスト：プレイ時の現在エネルギー全量を消費し、効果が X を参照

## エラーケース

| 条件 | 挙動 |
|------|------|
| 演出途中でのフェーズ遷移要求 | キューがアイドルになるまで遷移を保留 |
| ターン頭処理でプレイヤー死亡（自傷系） | 直ちに敗北判定（ドロー前でも） |
| 敵が全滅した直後の残処理 | 勝利確定後は以降のターン処理を打ち切り、CombatResult を返す |
| エネルギー不足でのカードプレイ | プレイ不可（input/cards 側でフィードバック） |

## 未対応ケース

- 追加ターン/行動順操作など特殊効果は段階対応
- 手札保持（Retain）等の細則は cards サブ項目と連携して実装
