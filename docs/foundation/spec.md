# foundation 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | アプリ起動 | LWJGL3 デスクトップランチャーでウィンドウを生成し、ゲームを初期化する |
| 2 | シーン管理 | メインメニュー／ゲーム画面などの Screen を切り替え、必要に応じてスタック管理する |
| 3 | 共通入力 | マウス座標・クリック・ドラッグ・キー入力を一元的に受け取り各画面へ供給する |
| 4 | アセット管理 | テクスチャ・フォント・音声を `AssetManager` でロード／参照／解放する |
| 5 | 仮想解像度描画 | 1920×1080 基準の Viewport で解像度非依存に描画し、リサイズ時はレターボックス |
| 6 | メインメニュー | 「新規ラン開始」「設定（任意）」「終了」を表示し操作を受け付ける |
| 7 | 配布 | Gradle ビルドと jpackage で Windows .exe を生成する手順を提供 |

## 画面・操作フロー

1. アプリを起動すると Boot 処理が走り、共通アセット（フォント・UI 画像・効果音）をロードする
2. ロード完了後、メインメニュー画面を表示する
3. ユーザが「新規ラン開始」を選択すると、ゲーム画面（本フェーズではプレースホルダ）へ遷移する
4. ゲーム画面で「中断／戻る」操作（ESC 等）を行うとメインメニューへ戻る
5. ユーザが「終了」を選択するとアプリを終了する

## 各画面 / 状態の詳細

### Boot（起動ロード）

- **表示内容**: ロゴまたはロード中表示（プレースホルダ可）。`AssetManager.update()` の進捗
- **操作**: なし（自動進行）
- **遷移先**: ロード完了でメインメニュー

### メインメニュー

- **表示内容**: タイトル、ボタン3種（新規ラン開始／設定（任意）／終了）。ホバー・押下のフィードバック
- **操作**: マウスホバーで強調、クリックで決定。キーボード上下＋Enter でも選択可
- **遷移先**: 新規ラン開始 → ゲーム画面 / 設定 → 設定画面（任意） / 終了 → アプリ終了

### ゲーム画面（プレースホルダ）

- **表示内容**: 本フェーズではダミー（背景＋「ゲーム画面」テキスト）。後続機能エリアが中身を実装
- **操作**: ESC で中断メニューまたはメインメニューへ戻る
- **遷移先**: メインメニュー

## エラーケース

| 条件 | メッセージ / 挙動 |
|------|------------------|
| アセットの読み込み失敗（ファイル欠落） | ログにエラー出力し、該当アセットはプレースホルダ（マゼンタのダミーテクスチャ／デフォルトフォント）で代替して継続 |
| ウィンドウ生成失敗（GPU/ドライバ） | 標準エラーへスタックトレースを出力して終了。ログにメッセージを残す |
| 極端なウィンドウサイズ（極小・超ワイド） | Viewport のレターボックス／ピラーボックスで UI 比率を維持し、はみ出しを防ぐ |

## 未対応ケース

- 設定項目の永続化（音量・解像度・フルスクリーン切替の保存）は任意・後続
- 多言語対応（本プロジェクトは日本語 UI を基本とし、ローカライズ機構は作らない）
- セーブデータの読み込みからの再開（`run` 機能エリアで扱う）
