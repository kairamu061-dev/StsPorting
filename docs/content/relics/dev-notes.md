# relics（content）開発メモ

## 実装上の判断

| 判断内容 | 理由 |
|----------|------|
| RelicManager が保持集合のフックを一括発火 | combat/run の各タイミングから1点呼び出しで全レリックに伝播。発火順は取得順 |
| 戦闘中効果は action-queue 経由、取得時は RunState 直接 | 戦闘の演出順を保ちつつ、メタ進行（最大HP等）は即時反映 |
| 「フック種類の代表」で初期選定 | onVictory/atBattleStart/onUseCard/onEquip 等を最小数で網羅し、量より仕組み |
| カウント系は境界でリセット | atBattleStart/atTurnStart で初期化し、状態の持ち越しバグを防ぐ |
