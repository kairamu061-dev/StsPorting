# cards（content）概要

## 目的・背景

アイアンクラッドおよび共通の具体カードを定義する。combat/cards が提供する仕組み（AbstractCard・piles・プレイのライフサイクル）の上に、各カードの効果（攻撃6/ブロック5 等）と数値・アップグレード差分を実装する。

全カード網羅はせず、「初期デッキ＋通常プレイが成立するコア報酬カード」を再現対象として選定する。選定はシステムの多様性（攻撃/ブロック/状態異常/ドロー/パワー/消滅）を代表できることを基準とする。

## スコープ

### 作るもの

- **初期デッキ**：ストライク×5、ディフェンド×4、バッシュ×1
- **コモン～アンコモンの代表カード**：ブロック獲得・状態異常付与・ドロー・複数ヒット・パワー等
- **代表的なパワーカード**：筋力獲得など POWER 種別の例
- **アップグレード差分**：各カードの + 版の数値/効果
- **CardLibrary**：CardId → AbstractCard 生成の登録

### 作らないもの

- 他キャラ専用カード
- レア/特殊カードの全網羅（段階的に追加）
- 呪い/状態異常カードの全種（代表のみ）

## スコープ補足（再現対象カードの初期リスト）

## 制約

- combat/cards の AbstractCard を継承し、`use()` で combat の GameAction/ApplyPowerAction を組む。
- 数値は原作準拠を目標。再現対象は段階的に増やせる（レジストリ登録）。

## 完了条件

- 初期デッキ 3 種が定義され、戦闘で正しく機能する（バッシュの脆弱付与含む）
- 代表報酬カードが定義され、報酬プールに供給できる
- 各カードのアップグレード差分が反映される
- カード効果のユニットテスト（ダメージ/ブロック/状態異常/ドロー）が通る
