# cards（content）開発メモ

## 実装上の判断

| 判断内容 | 理由 |
|----------|------|
| カードは base 値のみ持ち、修正は DamageAction 側 | 筋力/弱体/脆弱の適用を一箇所（combat）に集約し、カードは効果の組み立てに専念。原作の計算一元化に一致 |
| 初期リストは「システム多様性の代表」で選定 | 攻撃/全体/ブロック複合/状態異常/ドロー/一時&永続パワー/盤面依存 を最小数で網羅し、量より仕組みを再現 |
| POWER カードは付与後に消費（combat/cards の規約） | 原作仕様。content は ApplyPowerAction を積むだけ |
| Flex の一時筋力はターン終わりに戻すパワー/フックで表現 | 「一時筋力」を専用ロジックにせず powers の仕組みに載せる |
