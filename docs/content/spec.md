# content 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | カード定義 | ID・種別・コスト・効果（GameAction 列）・アップグレード・レアリティ |
| 2 | レリック定義 | ID・取得時/常時/戦闘フック・説明 |
| 3 | ポーション定義 | ID・効果・対象要否・レアリティ |
| 4 | 敵定義 | ID・HP レンジ・インテントパターン・行動 AI（確率/連続制限） |
| 5 | イベント定義 | ID・本文・選択肢・各選択の結果（状態変化） |
| 6 | エンカウンター定義 | ノード種別（通常/エリート/ボス）ごとの敵編成テーブル |
| 7 | レジストリ | 各定義を ID で登録・取得（combat/run/map から参照） |

## データ参照フロー

1. `run` の初期化が `content` から初期デッキ/初期レリックの ID を取得して RunState を構成
2. `map` がノード種別に応じて `content` のエンカウンター/イベント/報酬プールを参照
3. `combat` が `CombatRequest` の敵 ID とプレイヤーのカード ID を `content` から実体化して戦闘を構成
4. カード/レリック/パワーの効果は `combat` の GameAction/AbstractPower を組み合わせて表現

## 各データの詳細方針

### カード（content/cards）

- 初期デッキ：ストライク（攻撃6）、ディフェンド（ブロック5）、バッシュ（攻撃8＋脆弱2）
- 報酬コモン/アンコモン/レアの代表を選定（ブロック/状態異常/ドロー/パワーを網羅）
- アップグレード（+）は戦闘外（休憩鍛冶/一部効果）で適用。各カードに upgrade 差分を定義

### レリック（content/relics）

- 初期：バーニングブラッド（戦闘後 HP+6）
- 代表フック：ターン頭発動、被弾時、カードプレイ時、取得時即時 などを 1 つずつ以上

### ポーション（content/potions）

- 代表：回復/筋力/ブロック/敵デバフ など対象有無の両パターン

### 敵（content/monsters）

- 第 1 幕の通常敵（単体/複数編成）、エリート、ボスを各 1 体以上
- インテント：攻撃/防御/バフ/デバフ/複合 を網羅し、AI の連続行動制限を再現

### イベント（content/events）

- 選択肢ごとに「HP 増減/カード入手/レリック/金/呪い」等の結果を持つ代表イベント

## エラーケース

| 条件 | メッセージ / 挙動 |
|------|------------------|
| 未登録 ID を参照 | レジストリが例外/警告を出し、ログに ID を記録（実装途中の検知用） |
| 報酬プールが空（未実装種別） | `run` 側で報酬スキップ可能にする安全策と連携 |
| 敵 AI が選べる行動なし | フォールバック行動（無害な待機/基本攻撃）を定義 |

## 未対応ケース

- 全カード/敵/レリック/ポーション/イベントの網羅
- 他キャラ専用コンテンツ
- アセンション調整値

## サブ項目への分割

- `content/cards` / `content/relics` / `content/potions` / `content/monsters` / `content/events`
