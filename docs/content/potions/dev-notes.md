# potions（content）開発メモ

## 実装上の判断

| 判断内容 | 理由 |
|----------|------|
| 効果は action-queue/powers で表現 | カード/レリックと効果表現を統一し、演出順も揃える |
| needsTarget / combatOnly フラグを持たせる | 対象選択の要否・戦闘専用かを宣言的に扱い、使用フローを共通化 |
| 「対象有無＋種類の代表」で初期選定 | 自己バフ/ブロック/回復・敵デバフ/ダメージ・リソースを最小数で網羅 |
| 回復系は combatOnly=false | 戦闘外でも使えるポーションを表現 |
