# potions（content）概要

## 目的・背景

ポーション（即時使用の消費アイテム）を定義する。run のポーションスロットに保持され、戦闘中（一部は戦闘外）に使用して即時効果を発揮する。combat の action-queue／powers を用いて効果を表現する。

全ポーション網羅はせず、「対象有無の両パターン＋効果種類の代表」を再現対象とする。

## スコープ

### 作るもの

- **代表ポーション**：回復／筋力／ブロック／敵デバフ／ドロー／エネルギー などの代表
- **対象要否**：対象なし（自己/全体）と対象あり（単体敵）の両パターン
- **PotionLibrary**：PotionId → AbstractPotion 生成・登録
- **使用処理連携**：run のポーション使用（input 経由）から effect 発火・スロット消費

### 作らないもの

- 全ポーション網羅
- レア/特殊ポーションの全種
- ポーション画像（プレースホルダ）

## スコープ補足（再現対象の初期候補）

| ポーション | 対象 | 効果 |
|------------|------|------|
| 回復ポーション Block/Blood | 自己 | HP 回復（最大の一定割合 or 固定） |
| ブロックポーション | 自己 | ブロック+12 |
| 筋力ポーション Strength | 自己 | 筋力+2（戦闘中） |
| 弱体ポーション Weak | 単体敵 | 弱体3 付与 |
| 脆弱/Fear ポーション | 単体敵 | 脆弱3 付与 |
| エナジーポーション Energy | 自己 | エネルギー+2 |
| ドローポーション Swift | 自己 | 3 ドロー |
| 炎ポーション Fire | 単体敵 | 20 ダメージ |

> 「自己バフ/ブロック/回復」「敵デバフ/ダメージ」「リソース（エネ/ドロー）」で種類を代表。

## 制約

- combat の action-queue/powers で効果を表現（即時系も基本はアクション化）。
- 対象ありポーションは input の対象選択と連携。
- run のスロット上限（初期3）内で保持・使用・破棄。

## 完了条件

- 自己系（回復/ブロック/筋力/エネ/ドロー）が使用で即時に効く
- 敵対象系（弱体/脆弱/ダメージ）が対象選択して効く
- 使用でスロットが消費される
- ポーション効果のユニットテストが通る

## 画面イメージ

```
トップバー右:  [🧪Block][🧪Strength][ + ]   ← スロット3（空きは +）

```
