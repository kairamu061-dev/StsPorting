# content 開発メモ

## 実装上の判断

| 判断内容 | 理由 |
|----------|------|
| データをコードで定義（純 JSON でなく） | カード/敵の効果は GameAction 組み立てを伴い、原作の挙動を忠実に表現するにはふるまいと一体のコード定義が適切。型安全も得られる |
| レジストリ（ID→定義）で疎結合 | run/map/combat から ID 参照で取得。再現対象を段階的に増やせる |
| 「仕組みは完全・量は厳選」のコンテンツ側を担当 | プロジェクト方針。フック/インテントの代表を網羅しつつ総量を絞る |
| データ種別ごとにサブ項目分割 | カード/レリック/ポーション/敵/イベントは独立に定義・テスト可能 |
