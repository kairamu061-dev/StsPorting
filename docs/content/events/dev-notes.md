# events（content）開発メモ

## 実装上の判断

| 判断内容 | 理由 |
|----------|------|
| イベントをコード定義（分岐ロジック込み） | 条件付き選択肢・段階ページ・戦闘誘発など表現力が要るため、純データより適切 |
| 結果を EventOutcome（STAY/LEAVE/START_COMBAT/NEXT）で表現 | run/event 側の分岐を統一し、戦闘誘発や多段ページに対応 |
| 内容（content）と実行基盤（run/event）を分離 | UI/入力は run/event、データ/結果は content という分担 |
| 「結果パターンの代表」で初期選定 | リスク報酬/取引/報酬+代償/戦闘誘発を最小数で網羅 |
