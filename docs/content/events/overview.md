# events（content）概要

## 目的・背景

`?`（イベント）ノードで発生するイベントを定義する。イベントは本文と選択肢を提示し、選択に応じて RunState を変化させる（HP 増減・カード入手/削除・レリック・所持金・呪い付与・戦闘発生など）。run/event（実行基盤）が提示と入力を担い、本サブ項目は各イベントの内容（テキスト・選択肢・結果）を定義する。

全イベント網羅はせず、結果パターンの代表（リスク/報酬・取引・戦闘誘発など）を再現対象とする。

## スコープ

### 作るもの

- **代表イベント**：
  - リスク報酬型（HP を払ってレリック/カード）
  - 取引型（所持金やカードと引き換え）
  - 純報酬/純損失型（小さな祝福/呪い）
  - 戦闘誘発型（選択で戦闘へ）
- **イベント結果の表現**：RunState への変化（heal/loseHp/addCard/removeCard/addRelic/addGold/addPotion/呪い/戦闘開始）
- **EventLibrary**：EventId → AbstractEvent 生成・登録

### 作らないもの

- 全イベント網羅
- 複雑な多段分岐イベントの全種（段階対応）
- イベント画像（プレースホルダ）

## スコープ補足（再現対象の初期候補）

| イベント | タイプ | 概要 |
|----------|--------|------|
| ビッグフィッシュ Big Fish | リスク/報酬 | 餌の選択肢：最大HP増/全回復/レリック＋呪い |
| 取引商人/Dealer 系 | 取引 | 所持金やカードと引き換えに入手 |
| 呪われた壺/Cursed Tome 系 | リスク | HP を段階的に払いレリック、途中離脱可 |
| 黄金の偶像/Golden Idol | 報酬+リスク | レリック入手するが代償（呪い/罠ダメージ） |
| 眠る者/Sleeping 系 | 戦闘誘発 | 起こす=戦闘＋報酬 / 見逃す=何もなし |

## 制約

- run/event の実行基盤（提示・入力・結果適用）に乗る。本サブ項目は内容データと結果ロジック。
- 結果は RunState 操作 or 戦闘起動要求として表現。
- 量より「結果パターンの代表」を優先。

## 完了条件

- 代表イベントが提示され、選択で RunState が正しく変化する
- 戦闘誘発型が戦闘を起動し、戻り後に結果適用できる
- イベント結果のユニットテストが通る

## 画面イメージ

```
+-------------------------------------------+
|  [イベント名]                              |
|  本文テキスト...                            |
|                                           |
|  > 選択肢A（HP-8 → レリック入手）          |
|  > 選択肢B（何もせず立ち去る）              |
+-------------------------------------------+
```
