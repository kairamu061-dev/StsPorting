# 用語集

このプロジェクトで使用するドメイン固有の用語を定義する。
実装・設計・仕様書内で用語を使う際はここに定義されたものに統一する。

---

| 用語 | 定義 |
|------|------|
| STS | Slay the Spire（原作）の略。本プロジェクトの再現対象 |
| ラン (Run) | ゲーム開始から敗北/クリアまでの 1 回のプレイ。永続状態を持ち越す単位 |
| RunState | 1 ランで持ち越すプレイヤー永続状態（HP/デッキ/レリック/ポーション/金/位置/シード） |
| 幕 (Act) | マップの大区分。1 幕ごとにノードマップが生成され、最上段にボスがいる |
| フロア (Floor) | ラン内で進んだノードの通し段数 |
| ノード (Node) | マップ上の到達地点。種別＝モンスター/エリート/休憩/ショップ/宝箱/イベント/ボス |
| エンカウンター (Encounter) | モンスター/エリート/ボスノードに割り当てられる敵編成 |
| 戦闘 (Combat) | ターン制カードバトル。プレイヤーと敵が交互に行動する |
| アクションキュー (Action Queue) | 戦闘中の全処理を逐次実行する原作準拠の行動キュー（GameActionManager 相当）。解決の順序/タイミングの根幹 |
| GameAction | アクションキューで解決される処理単位（ダメージ/ブロック/ドロー/パワー付与など） |
| 割り込み (addToTop) | 実行中のアクションがキュー先頭へ別アクションを挿入し、先に解決させる仕組み |
| カード (Card) | 攻撃/スキル/パワーの 3 種別を持つプレイ単位。コスト・対象・効果・アップグレードを持つ |
| 山札/手札/捨て札/消滅札 | 戦闘中のカードの所在。ドロー元/保持/使用後/除外。山札切れ時は捨て札を再シャッフル |
| エネルギー (Energy) | カードプレイのための毎ターン回復する資源。未使用分は繰り越さない |
| ブロック (Block) | ダメージを軽減する一時値。原則ターン開始時に消える |
| パワー (Power) | クリーチャーに付与されるバフ/デバフ（筋力/弱体/脆弱/中毒 等）。スタック・発動タイミングを持つ |
| 筋力 (Strength) | 攻撃ダメージに加算されるバフ |
| 弱体 (Weak) | 攻撃ダメージを 25% 減少させるデバフ |
| 脆弱 (Vulnerable) | 受ける攻撃ダメージを 50% 増加させるデバフ |
| インテント (Intent) | 敵が次ターンに行う行動の予告表示（攻撃/防御/バフ/デバフ等） |
| クリーチャー (Creature) | プレイヤー/敵の共通基底（HP/ブロック/パワーを持つ） |
| レリック (Relic) | 常時/取得時/戦闘中に効果を発揮する永続アイテム |
| ポーション (Potion) | 即時使用の消費アイテム。スロット上限内で所持 |
| 報酬 (Reward) | 戦闘勝利後に得る金/カード選択/レリック/ポーション |
| アイアンクラッド (Ironclad) | 本プロジェクトで再現する唯一のキャラクター |
| 仮想解像度 | レイアウト基準の論理解像度（1920×1080）。実ウィンドウへは Viewport でレターボックス表示 |
| 決定的 RNG | シードから派生し、同一シードで同一結果になる乱数。再現性とテストのため系統別に分離 |
