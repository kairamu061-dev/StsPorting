# map 概要

## 目的・背景

1 つの幕（Act）のマップを生成し、プレイヤーが進路を選んでノードへ向かう画面を担う機能エリア。
原作の「複数の経路に分岐したノードグラフを下から上へ登る」構造を再現する。マップの見た目・経路選択の操作感も再現対象。

`run` の `RunController` がマップを介してノードを選ばせ、選ばれたノードの種別を受け取って各画面へルーティングする。

## スコープ

### 作るもの

- **マップ生成**：1 幕分のノードグラフ（行＝フロア、列＝経路）。原作準拠の生成則（各行のノード数、経路の接続、ノード種別の配分、休憩/ショップ/エリート/宝箱の出現規則、ボスは最上段に単一）
- **ノード種別の配置**：Monster / Elite / Rest / Merchant / Treasure / Event / Boss の割り当て規則（決定的 RNG）
- **マップ描画**：ノードアイコン、経路線、現在位置、到達可能ノードの強調
- **経路選択の操作**：選べるノードのみクリック可、ホバーで強調、選択で `run` へ通知
- **進行制約**：直前ノードから線で繋がった次行ノードのみ選択可能

### 作らないもの

- ノードに入った後の中身（戦闘=`combat`、休憩/ショップ/宝箱/イベント=`run` サブ項目）
- 複数幕をまたぐ全体地図（1 幕単位で生成。幕送りは `run` が管理）
- マップ上のスクロール演出など装飾的要素（最小限の再現に留める）

## 制約

- foundation（描画/入力）と `run`（RunState の現在位置・シード）に依存。
- 生成は決定的（同一シードで同一マップ）。ヘッドレスで生成則をユニットテスト可能にする。
- 原作の生成アルゴリズムを完全一致させるのは段階目標とし、まず「妥当な分岐マップが規則的に生成され選択進行できる」ことを優先。

## 完了条件

- 幕開始時にノードグラフが生成され、最下行から開始できる
- 現在位置から線で繋がった次行ノードのみ選択でき、選ぶと `run` に種別が渡る
- 同一シードで同一マップが再現される
- マップ生成則のユニットテストがヘッドレスで通る

## 画面イメージ

```
        [BOSS]              ← 最上段（ボス）
       /  |   \
   [M] [?] [R]              ← 各行に複数ノード・経路で接続
    |  X   |  \
   [M] [E] [$] [T]
     \  |  /  |
   [M] [M] [?] ...
        |
    [現在地]                ← 最下段から開始
```

凡例: M=モンスター E=エリート R=休憩 $=ショップ T=宝箱 ?=イベント
