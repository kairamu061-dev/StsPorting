# map 開発メモ

## 実装上の判断

| 判断内容 | 理由 |
|----------|------|
| map を分割しない | 生成/描画/選択が密結合の単一画面で、単体検証が困難な小規模単位。分割ルールの例外に該当 |
| 生成を決定的（シード固定）に | 同一シードで同一マップを保証し、生成則をユニットテスト可能にする |
| 生成後に到達性検証＋再生成 | 孤立ノードや到達不能経路を防ぎ、必ず最下→ボスの経路を保証する |
| 原作生成則は段階的に近づける | まず「妥当な分岐マップで進行できる」ことを優先し、確率・制約の完全一致は後段で詰める |

## 今後の課題

- 原作のマップ生成アルゴリズム（経路の引き方・種別配分の確率）との差分を詰める
- 複数幕対応（幕ごとのボス・種別配分の差）
- マップ上の特殊/隠しノードは段階対応

## ユーザへの要望

- マップの見た目（ノードアイコン・経路線）をどこまで原作に寄せるか。アセット非同梱のため初期は簡易図形で進め、後で差し替える方針で問題ないか確認したい
