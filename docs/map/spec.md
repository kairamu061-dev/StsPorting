# map 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | マップ生成 | 1 幕分のノードグラフを決定的に生成（行・列・経路接続） |
| 2 | ノード種別割当 | 各ノードへ種別を規則的に配分（出現規則・制約） |
| 3 | マップ描画 | ノードアイコン・経路線・現在位置・選択可能ノードの強調 |
| 4 | 経路選択 | 接続された次行ノードのみ選択可。選択を run へ通知 |
| 5 | 進行更新 | 選択後に現在位置を更新し、到達可能集合を再計算 |

## 画面・操作フロー

1. 幕開始時、`run` がシードと共にマップ生成を要求
2. マップ画面を表示（最下行が開始位置、未選択）
3. プレイヤーは最下行のいずれか（または現在位置の次行で接続されたもの）を選択
4. 選択ノードの種別を `run` に返し、`run` が対応画面を起動
5. ノード解決後にマップへ戻ると、現在位置から繋がる次行ノードのみ選択可能
6. 最上段のボスノードに到達・選択でボス戦へ → クリアで次幕（または勝利）

## 各画面 / 状態の詳細

### マップ画面

- **表示内容**: ノードグラフ全体、種別アイコン、経路線、現在位置マーカー、選択可能ノードの発光/拡大
- **操作**:
  - 選択可能ノードをホバー → 強調
  - クリック → そのノードへ進む（`run` に種別通知）
  - 選択不可ノード（接続なし/現在行以外）はクリック無効
- **遷移先**: 選択した種別の画面（戦闘/休憩/ショップ/宝箱/イベント/ボス）

## マップ生成規則（再現方針）

原作準拠を目標に、段階的に近づける。初期実装の規則:

- 1 幕は約 15 行（最下=開始域、最上=ボス）
- 各行に 2〜5 ノード。隣接行間を経路（エッジ）で接続し、交差や合流を許容
- ボス行は単一ノード。直前行はボスへ集約
- 休憩所は最終行手前付近に出やすい、ショップ/宝箱/エリートには出現フロア制約や最低距離制約を設ける
- 同一シードで同一マップ（決定的）

> 詳細な確率・制約（エリート最低フロア、連続同種抑制など）は design.md と dev-notes で管理し、原作差分を順次詰める。

## エラーケース

| 条件 | メッセージ / 挙動 |
|------|------------------|
| 接続のないノードをクリック | 無効（フィードバックのみ、状態変化なし） |
| 生成で孤立ノード/到達不能が発生 | 生成則で必ず最下→最上の経路が存在するよう保証（生成後に検証し、満たさなければ再生成） |
| ボス到達時 | ボス戦を起動。クリアで `run` が次幕生成 or 勝利判定 |

## 未対応ケース

- 複数幕にまたがる地図表示（1 幕単位）
- マップ上の隠し/特殊ノード（段階対応）
- スクロール/パララックス等の装飾演出
