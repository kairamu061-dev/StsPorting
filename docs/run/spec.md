# run 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | ラン初期化 | アイアンクラッドの初期 HP/デッキ/レリックで RunState を生成し、シードを確定する |
| 2 | ノード解決ループ | マップで選んだノードの種別に応じて対応画面を起動し、結果を RunState に反映 |
| 3 | トップバー HUD | HP・所持金・レリック・ポーション・フロアを常時表示（戦闘外の各画面で共通） |
| 4 | 戦闘報酬 | 勝利後にカード3択・所持金・（条件付きで）ポーション/レリックを提示し選択を反映 |
| 5 | ポーション所持管理 | スロット上限（初期3）でポーションを保持・使用・破棄 |
| 6 | レリック取得 | レリックを RunState に追加し、取得時効果（即時系）をフック |
| 7 | ゲームオーバー判定 | 戦闘結果が敗北（HP0）ならゲームオーバー画面へ |
| 8 | 勝利判定 | 規定の最終ボス撃破で勝利画面へ |
| 9 | RNG 管理 | シードから系統別（マップ/カード報酬/モンスター/ポーション/イベント等）の乱数列を供給 |

## 画面・操作フロー

1. メインメニューで「新規ラン開始」→ RunState を初期化（後述の初期値）
2. マップ画面に入り、選択可能なノードから 1 つ選ぶ
3. ノード種別ごとに画面を起動：
   - Monster / Elite / Boss → 戦闘（`combat`）
   - Rest → 休憩所 / Merchant → ショップ / Treasure → 宝箱 / Event → イベント
4. 戦闘勝利時は報酬画面 → 選択完了でマップへ戻る
5. 戦闘敗北時はゲームオーバー画面 → メインメニューへ
6. ノード解決のたびにフロアを進め、最終ボス撃破で勝利画面
7. トップバー HUD は戦闘外の各画面で常時表示し、ポーション使用などの操作を受け付ける

## 各画面 / 状態の詳細

### ラン初期化（RunState）

- **初期値（アイアンクラッド）**:
  - 最大 HP 80 / 現在 HP 80
  - 所持金 99
  - 初期デッキ：ストライク×5、ディフェンド×4、バッシュ×1、（再現可能なら）バーニングブラッド（初期レリック）
  - 初期レリック：バーニングブラッド（戦闘後 6 回復）
  - ポーションスロット：3（空）
  - フロア 0 / 第 1 幕
- **シード**: 数値シード 1 つから派生して系統別 RNG を生成（決定的）

### ノード解決ループ（RunController）

- **表示内容**: 現在のノード種別に応じた子画面。上部にトップバー HUD
- **操作**: 子画面に委譲。子画面終了時に `NodeResult` を返す
- **遷移先**: 結果に応じてマップ／報酬／ゲームオーバー／勝利

### 戦闘報酬

- **表示内容**: 獲得所持金（10〜20 程度の乱数、原作準拠範囲）、カード3択（スキップ可）、エリート/ボスはレリック、確率でポーション
- **操作**: カードを1枚選択またはスキップ、ポーション/レリックを受け取る
- **遷移先**: 受け取り完了でマップへ

### ゲームオーバー / 勝利

- **表示内容**: 到達フロア・撃破数などの簡易サマリ（任意）。再挑戦/メニューへ
- **操作**: 「メインメニューへ」
- **遷移先**: メインメニュー

## エラーケース

| 条件 | メッセージ / 挙動 |
|------|------------------|
| ポーションスロット満杯で取得 | 取得不可とし、破棄または「いっぱいです」表示（原作は捨てさせる UI）。本実装は取得スキップ可 |
| カード報酬の対象プールが空 | 報酬カードを出さずスキップ可能にする（コンテンツ未実装分の安全策） |
| 子画面が未実装ノードを返す | ログ警告し、状態を変えずマップへ戻す（段階的実装中の安全策） |
| RunState 不整合（HP が最大超過等） | 取得/更新時に 0〜最大でクランプ |

## 未対応ケース

- 中断データからの再開 UI（保存形式は設計に含めるが UI は任意・後続）
- イベント中の複雑な分岐の全網羅（`content` 側で再現対象を絞る）
- 第 2 幕以降・心臓（最終真ボス）まで含めるかは段階実装（まず第 1 幕の完走を最小目標とする）

## サブ項目への分割

独立した UI 画面を持つため、以下をサブ項目に分割する（`/add-feature run/<name>`）。`run` 親は RunState・RunController・RNG・トップバー HUD・報酬という横断的責務を担う。

- `run/rest-site`（休憩所：休息/鍛冶）
- `run/shop`（ショップ：購入/カード削除）
- `run/treasure`（宝箱）
- `run/event`（イベント実行基盤）
