# treasure（宝箱）開発メモ

## 実装上の判断

| 判断内容 | 理由 |
|----------|------|
| 中身は treasure 系統 RNG で決定的生成 | 同一シードで同一中身。テスト可能 |
| 既所持レリックは除外して抽選 | 重複取得を防ぐ原作挙動 |
| ポーションはスロット空き時のみ | run のポーション管理に従い破綻を防ぐ |
| 「立ち去る」分岐口を残す | 罠/呪い付き宝箱の段階対応の余地 |
