# event（イベント実行基盤）概要

## 目的・背景

`?`（イベント）ノードの**実行基盤**を担う。イベントの本文・選択肢を提示し、入力を受けて選択結果を適用、結果に応じてマップ/続きページ/戦闘へ分岐する。イベントの**内容データ**は content/events が定義し、本サブ項目はそれを駆動する画面・進行ロジックを提供する（仕組みと内容の分担）。

## スコープ

### 作るもの

- **イベント画面**：本文表示、選択肢ボタン（条件で非活性）、（必要なら）ページ送り
- **選択適用**：選んだ EventOption の `apply(RunState)` を実行し EventOutcome を取得
- **結果分岐**：STAY（続き表示）/ LEAVE（マップへ）/ START_COMBAT（戦闘起動→戻りで残り適用）/ NEXT（別ページ）
- **イベント抽選**：イベントノードで content/events のプールから1件を RunRng で決定的に選ぶ
- **カード選択 UI 連携**：結果がカード入手/削除を要求する場合の選択
- **戦闘誘発の往復**：START_COMBAT 後、CombatResult を受けて続き処理

### 作らないもの

- 各イベントの本文/選択肢/結果（content/events）
- 戦闘そのもの（combat）

## 制約

- foundation の画面基盤・run の RunState/RunController に依存。
- イベント抽選は決定的（同一シードで同一イベント）。
- 結果適用は content/events の apply に委譲し、基盤は分岐と UI に専念。

## 完了条件

- イベントが本文・選択肢を提示し、条件で選択肢が非活性になる
- 選択で RunState が変化し、EventOutcome に応じて正しく分岐する
- 戦闘誘発イベントが戦闘を起動し、戻り後に残り処理を行いマップへ戻る
- イベント抽選が決定的に動く

## 画面イメージ

```
+-------------------------------------------+
|  [イベント名]                              |
|  本文テキスト...                            |
|                                           |
|  > 選択肢A                                 |
|  > 選択肢B（条件不足: 非活性）             |
+-------------------------------------------+
選択 → apply → outcome で分岐（マップ/続き/戦闘）
```
