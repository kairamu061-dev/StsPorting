# event（イベント実行基盤）開発メモ

## 実装上の判断

| 判断内容 | 理由 |
|----------|------|
| 実行基盤（run/event）と内容（content/events）を分離 | UI/入力/分岐は基盤、本文/選択肢/結果は content。仕組みと内容の分担を一貫させる |
| 結果を EventOutcome で受け基盤が分岐 | 戦闘誘発・多段ページ・退出を統一的に扱う |
| 「立ち去る」既定肢を必ず用意 | 全選択肢非活性での詰みを防ぐ |
| 戦闘誘発は controller.startCombat のコールバックで往復 | combat 境界（CombatRequest/Result）に乗せ、戻り後に残り処理 |
